## 6. Remaining non-blocking gaps

These still exist, but they did not block the minimal PIC-MCC run:

- `Tools/FieldSolver/src/MultiBoundaryElectricFieldSolver.cpp`
  - current multigrid path is now a practical Jacobi-smoothed fallback, not a full geometric multigrid implementation
  - periodic constraints can now auto-pair axis-aligned faces, oblique translated parallel faces, and rotated congruent faces, but still do not reconstruct arbitrary curved or non-isomorphic periodic topology
  - floating conductors are now handled as equipotential node groups, but still use a soft reference instead of a fully coupled charge-conservation unknown

- `Tools/PICcore/src/PICCycle.cpp`
  - legacy boundary recovery is now limited to small near-face overshoots with a valid local element hint instead of acting like a general escape catch-all

Notes:

- The current minimal PIC-MCC path now uses `Tools/FieldSolver/src/PoissonSolver.cpp` as the preferred runtime field solve path.
- `Tools/PICcore/src/PICCycle.cpp` no longer carries a layered 1D Poisson compatibility solve.
- The main remaining solver-quality gap is improving the multiboundary solver from a usable fallback into a fully featured multigrid/boundary-coupled implementation.

## 8. Recommended next refactor order

1. Upgrade `MultiBoundaryElectricFieldSolver` from the current usable fallback to a true multigrid / coupled-boundary implementation.
2. Extend periodic pairing beyond congruent face matching to support more general curved and non-isomorphic surface topology.
3. Replace more of the remaining legacy particle-loss heuristics in `Tools/PICcore/src/PICCycle.cpp` once runtime boundary configurations are validated on more cases.
4. Migrate only the `ref/material` pieces needed by the current case.
5. Leave `ref/ui`, `ref/visualization`, and `ref/python` until the physics chain is stable.
